# T-SEC-01 — NATS TLS Validation

## Problem
В текущей конфигурации NATS/JetStream данные передаются в открытом виде (plaintext). Для обеспечения безопасности в продакшене (Internal Security) необходимо использовать TLS для шифрования трафика между Router и NATS.

## Goal
Валидировать возможность работы Router с NATS через защищенное соединение (TLS):
- Сгенерировать необходимые сертификаты (CA, Server, Client).
- Настроить NATS Server на использование TLS.
- Настроить Erlang-клиент (enats) на использование TLS.
- Доказать успешность подключения через тесты.

## Outcome
1. Набор скриптов для генерации сертификатов в `_artifacts/certs/`.
2. Конфигурационный файл NATS с поддержкой TLS.
3. Обновленные конфиги Router для работы через TLS.
4. Успешный прогон `nats_status.sh` (или аналога) подтверждающий шифрование.
